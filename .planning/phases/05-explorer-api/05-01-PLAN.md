# Phase 5 Plan 1: Explorer API

## Objective

Add explorer API endpoints to the existing dashboard FastAPI app for querying security events, epoch history, node history, overwatch audit logs, and search functionality. These endpoints power the network explorer view.

## Context

The dashboard REST API already exists at `subnet/dashboard/rest_api.py` with endpoints for nodes, events, topology, and overwatch. The data is stored in RocksDB using the nmap pattern (e.g., `nmap:heartbeat:epoch:peer_id`, `nmap:tee_quote:epoch:peer_id`, `nmap:mock_work:epoch:peer_id`).

This plan extends the existing dashboard API with explorer-specific endpoints that provide richer filtering, epoch-level summaries, per-node history, overwatch audit logs with tamper evidence, and cross-entity search.

## Tasks

### Task 1: Security event model and indexer
type="auto"

Create a security event model and in-memory indexer that aggregates data from existing RocksDB nmaps (heartbeat, tee_quote, mock_work) into a queryable format with event types and severity levels.

**Files:**
- `subnet/dashboard/explorer/models.py` - SecurityEvent, EpochSummary, AuditLogEntry Pydantic models
- `subnet/dashboard/explorer/__init__.py` - Package init
- `subnet/dashboard/explorer/indexer.py` - EventIndexer that reads from RocksDB and provides filtered queries

**Done when:** Models are defined, indexer can read from RocksDB nmaps and return filtered SecurityEvent lists.

### Task 2: Explorer API endpoints (EXP-01, EXP-02, EXP-03, EXP-04, EXP-05)
type="auto"

Create the explorer router with all five endpoint groups:
- `GET /api/explorer/events` - Query security events with filters (EXP-01)
- `GET /api/explorer/epochs` - Epoch history summaries (EXP-02)
- `GET /api/explorer/epochs/{epoch}` - Single epoch detail (EXP-02)
- `GET /api/explorer/nodes/{peer_id}/history` - Node event history (EXP-03)
- `GET /api/explorer/audit` - Overwatch audit log with tamper evidence (EXP-04)
- `GET /api/explorer/search` - Search by peer_id, epoch, event_type, hardware_id (EXP-05)

**Files:**
- `subnet/dashboard/explorer/router.py` - FastAPI router with all endpoints
- `subnet/dashboard/ws_bridge.py` - Mount the explorer router

**Done when:** All endpoints are implemented and mounted in the dashboard app.

### Task 3: Tests for explorer API
type="auto"

Write comprehensive tests for all explorer endpoints using FastAPI TestClient and a mock RocksDB.

**Files:**
- `tests/dashboard/__init__.py`
- `tests/dashboard/test_explorer_api.py` - Tests for all 6 endpoints with filter combinations

**Done when:** All tests pass, covering all endpoints and filter combinations. Existing 400 tests still pass.
