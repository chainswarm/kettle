# Phase 9 Plan 1: Agent Integration & Deployment

## Objective

Create an x402 payment middleware layer that wraps the existing Frontier gateway, enabling autonomous agents to pay-per-request for inference via the HTTP 402 protocol. Add Docker Compose service and example agent client.

## Context

- @subnet/frontier/app.py - Existing OpenAI-compatible gateway with capacity routing
- @subnet/frontier/capacity.py - Capacity table for node routing
- @subnet/frontier/cli.py - Existing frontier CLI
- @docker-compose.tee-dev.yml - Dev stack to add x402-frontier service

## Tasks

### Task 1: Create x402 payment models and config
type="auto"

Create the x402 module with Pydantic models for payment headers, pricing config, and payment verification.

**Files:**
- subnet/x402/__init__.py
- subnet/x402/models.py - PaymentRequired response, PaymentHeader, PricingConfig, SettlementReceipt
- subnet/x402/config.py - X402Config with pricing tiers, wallet address, network settings

**Done when:** Models importable, config loads from env vars.

### Task 2: Create x402 payment middleware
type="auto"

Create FastAPI middleware that intercepts requests, checks for x402 payment headers, validates payment, and returns 402 Payment Required with pricing info when no valid payment is present.

**Files:**
- subnet/x402/middleware.py - X402PaymentMiddleware class
- subnet/x402/verification.py - Payment verification logic (signature check, amount validation)

**Done when:** Middleware can be applied to FastAPI app, returns 402 with proper headers on unpaid requests, passes through paid requests.

### Task 3: Create x402 frontier app and CLI
type="auto"

Create the x402-wrapped frontier app that combines the existing frontier gateway with x402 payment middleware. Add a CLI entry point.

**Files:**
- subnet/x402/app.py - create_x402_app() factory wrapping frontier
- subnet/x402/cli.py - CLI entry point for x402 frontier
- Update pyproject.toml with new script entry

**Done when:** `run_x402_frontier` CLI entry point works, app serves /v1/chat/completions with x402 layer.

### Task 4: Add Docker Compose service for x402 frontier
type="auto"

Add x402-frontier service to docker-compose.tee-dev.yml.

**Files:**
- docker-compose.tee-dev.yml - Add x402-frontier service

**Done when:** `docker compose -f docker-compose.tee-dev.yml config` validates with x402-frontier service.

### Task 5: Create example agent client
type="auto"

Create an example Python script showing how an autonomous agent uses the x402 frontier with payment.

**Files:**
- examples/x402-agent-client/README.md
- examples/x402-agent-client/agent_client.py
- examples/x402-agent-client/requirements.txt

**Done when:** Example script is complete and documented.

### Task 6: Add integration tests for full 402 flow
type="auto"

Test the complete flow: request -> 402 -> payment -> inference -> settlement.

**Files:**
- tests/x402/test_models.py
- tests/x402/test_middleware.py
- tests/x402/test_app.py
- tests/x402/test_integration.py

**Done when:** All tests pass, covering the full 402 payment flow.
