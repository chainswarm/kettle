# Phase 6 Plan 01: Explorer UI

**Objective:** Add epoch timeline, node history, overwatch audit log, and search to the existing Vue.js dashboard. Wire views to the existing `/api/*` REST endpoints. Integrate with existing navigation and Tailwind theme.

## Context

The dashboard already has:
- ExplorerView (node grid + event feed + topology)
- NodeDetailView (heartbeat, TEE, hardware, activity)
- AdminView (DB stats, overwatch feed, event inspector)
- AppHeader with nav (Explorer, Admin/Login, connection status)
- Network store with fetchNodes, fetchNodeDetail, fetchTopology, fetchOverwatch
- Types: SubnetNode, DHTEvent, TopologyData, OverwatchEvent, NodeDetail, DbStats
- Backend REST: /api/nodes, /api/nodes/:peer_id, /api/events, /api/topology, /api/overwatch

New views needed:
- Epoch Timeline: browse epochs, see events grouped by epoch (uses /api/events with all topics)
- Node History: full security event timeline for a single node (uses /api/nodes/:peer_id + /api/events filtered)
- Overwatch Audit Log: dedicated page for slash events and tamper detections (uses /api/overwatch)
- Search: global search bar in header (peer_id, epoch number, event type filtering)

## Tasks

### Task 1: Add explorer TypeScript types and API composable
type="auto"

Create a composable `useExplorerApi` for fetching explorer data, and add any missing types.

**Actions:**
- Add `ExplorerEpoch`, `ExplorerEvent`, `SearchResult` types to `types/index.ts`
- Create `composables/useExplorerApi.ts` with functions:
  - `fetchEpochs(page, limit)` - fetches events grouped by epoch from /api/events
  - `fetchEpochEvents(epoch)` - fetches all events for a specific epoch
  - `fetchNodeHistory(peerId)` - fetches node detail + filtered events
  - `fetchAuditLog(filter?)` - fetches overwatch events with optional filtering
  - `search(query)` - searches across nodes, epochs, events

**Done:** Types compile, composable exports all functions.

### Task 2: Epoch Timeline view
type="auto"

Create `views/EpochTimelineView.vue` - browse epochs with event summary per epoch.

**Actions:**
- Shows paginated list of epochs (newest first)
- Each epoch card shows: epoch number, event count by type, participating nodes
- Click epoch to expand and see individual events
- Pagination controls (prev/next)
- Loading and empty states
- Uses existing theme (green-50, green-200 borders, white bg)

**Done:** View renders, shows epoch data grouped correctly.

### Task 3: Node History view
type="auto"

Create `views/NodeHistoryView.vue` - full security event timeline for a node.

**Actions:**
- Header with node ID, status badge, TEE score
- Chronological event timeline (heartbeats, TEE quotes, work records, overwatch)
- Each event shows: type badge, epoch, timestamp, key data
- Filter by event type (toggleable chips)
- Link back to node detail
- Uses existing StatusBadge component

**Done:** View renders node history with filterable timeline.

### Task 4: Overwatch Audit Log view
type="auto"

Create `views/AuditLogView.vue` - dedicated page for slash events and tamper detections.

**Actions:**
- Table/list of all overwatch events from /api/overwatch
- Columns: epoch, node (peer_id truncated), result (pass/fail/tampered), details
- Filter by result type (pass, fail, tampered)
- Color-coded rows (green for pass, red for fail/tampered)
- Summary stats at top (total audits, failures, tamper detections)
- Uses existing OverwatchEvent type and color scheme

**Done:** View renders audit log with filtering and stats.

### Task 5: Search bar component and integration
type="auto"

Add search bar to AppHeader that searches across peer IDs, epoch numbers, and event types.

**Actions:**
- Create `components/SearchBar.vue` with:
  - Text input with search icon
  - Dropdown results panel (shows as you type, debounced)
  - Result categories: Nodes, Epochs, Events
  - Click result navigates to appropriate view
- Integrate into AppHeader between nav links and status indicator
- Search logic: match peer_id prefix, epoch number, event type name

**Done:** Search bar appears in header, typing shows results, clicking navigates.

### Task 6: Router and navigation integration
type="auto"

Wire all new views into the router and add nav items to AppHeader.

**Actions:**
- Add routes: /epochs, /epochs/:epoch, /node/:peerId/history, /audit-log
- Add nav links in AppHeader: Epochs, Audit Log (between Explorer and Admin)
- Ensure all routes work with browser back/forward
- Verify existing routes still work

**Done:** All nav items visible, routes resolve correctly, no regressions.
