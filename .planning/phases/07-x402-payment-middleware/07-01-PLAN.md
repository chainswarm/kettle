# Phase 7 Plan 01: x402 Payment Middleware

## Objective

Implement x402 HTTP payment protocol middleware for the frontier inference gateway. When a client
hits a paid endpoint without payment, the middleware returns HTTP 402 with payment requirements.
Clients attach payment headers; the middleware verifies payment via the x402 facilitator API
before forwarding the request.

## Context

- Existing frontier gateway: `subnet/frontier/app.py` (FastAPI, OpenAI-compatible)
- x402 protocol: HTTP 402 Payment Required with structured JSON payment requirements
- Facilitator API: External service that verifies and settles payments
- Scheme: "upto" EVM — client signs an ERC-20 permit authorizing up to X tokens

## Tasks

### Task 1: Pydantic models and configuration
type="auto"

Create `subnet/x402/` module with Pydantic models for x402 protocol objects and
environment-driven configuration.

**Files:**
- `subnet/x402/__init__.py`
- `subnet/x402/models.py` — PaymentRequired, PaymentPayload, SettlementResponse, PaymentRequiredInfo
- `subnet/x402/config.py` — X402Config with env vars (X402_RECEIVER_ADDRESS, X402_FACILITATOR_URL, X402_NETWORK, X402_ASSET_ADDRESS, X402_MAX_AMOUNT_REQUIRED, X402_RESOURCE, X402_DESCRIPTION, X402_MIME_TYPE, X402_CDP_API_KEY_ID, X402_CDP_API_KEY_SECRET)

**Done when:**
- Models importable from subnet.x402.models
- Config loads from environment variables with sensible defaults
- Pydantic validation works for all fields

### Task 2: Facilitator client
type="auto"

Create an httpx-based client that calls the x402 facilitator API to verify and settle payments.

**Files:**
- `subnet/x402/facilitator.py` — FacilitatorClient with verify() and settle() methods

**Done when:**
- verify() sends POST to facilitator /verify endpoint with payment payload
- settle() sends POST to facilitator /settle endpoint
- Proper error handling for network failures, non-200 responses
- Returns typed SettlementResponse

### Task 3: x402 middleware
type="auto"

Create FastAPI middleware that intercepts requests to protected routes:
1. If no `X-PAYMENT` header: return 402 with PaymentRequired body
2. If `X-PAYMENT` header present: decode, verify via facilitator, settle, then proceed

**Files:**
- `subnet/x402/middleware.py` — X402PaymentMiddleware class

**Done when:**
- Middleware can be mounted on any FastAPI app
- Returns proper 402 response with payment requirements JSON
- Verifies payment via facilitator before forwarding
- Passes through on successful settlement
- Handles facilitator errors gracefully (502 on facilitator failure)

### Task 4: Integration with frontier
type="auto"

Wire the x402 middleware into the frontier app factory so it can be optionally enabled
via configuration.

**Files:**
- `subnet/frontier/app.py` — add optional x402 middleware integration
- `subnet/x402/config.py` — add X402_ENABLED flag

**Done when:**
- create_app accepts optional x402_config parameter
- When X402_ENABLED=true, middleware is active on /v1/chat/completions
- When disabled (default), no behavior change
- Health endpoint never requires payment
