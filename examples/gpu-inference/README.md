# GPU Inference Example — TEE-Attested LLM via NVIDIA NIM

Example subnet implementation that runs LLM inference on a confidential GPU (NVIDIA H100) and proves the output was generated by genuine, unmodified code inside a TEE.

## Architecture

```
GPU VM (A10 for testing / H100 for confidential)
┌─────────────────────────────────────────────────────┐
│                                                     │
│  ┌───────────────────────────┐                      │
│  │  NVIDIA NIM               │                      │
│  │  Llama 3.2 1B Instruct    │ ← GPU                │
│  │  localhost:8000            │                      │
│  └──────────┬────────────────┘                      │
│             │ OpenAI API                             │
│  ┌──────────▼────────────────┐                      │
│  │  Miner (TEE subnet node)  │                      │
│  │  1. Receive challenge     │                      │
│  │  2. Call NIM for inference │                      │
│  │  3. Sign output (RA-TLS)  │ ← CPU (mock or TEE)  │
│  │  4. Publish to DHT        │                      │
│  └───────────────────────────┘                      │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## What it proves

| Property | How |
|----------|-----|
| Code is genuine | TEE attestation (MRENCLAVE/measurement) |
| Output is from this enclave | Signed with RA-TLS session key (F-02 cert pubkey binding) |
| Inference actually ran | NIM returns token count + latency; validator checks non-empty output |
| GPU is real (H100 only) | NVIDIA GPU attestation via `nv-attestation-sdk` (optional, see §10c in anti-cheat) |

| VM | GPU | Price/hr | Confidential? | Use case |

The **A10 VM** is recommended for initial testing — it validates the NIM integration pipeline
at minimal cost. TEE attestation is already proven by the 260 existing in-memory tests.

## Prerequisites

1. **Azure GPU VM** (A10 for testing, or H100 for confidential GPU)
2. **NVIDIA Container Toolkit** installed
3. **NGC API key** from https://ngc.nvidia.com/

```bash
# Install NVIDIA Container Toolkit
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
  sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker

# Login to NVIDIA NGC registry
echo $NGC_API_KEY | docker login nvcr.io -u '$oauthtoken' --password-stdin
```

## Quick start

```bash
# Set your NGC API key
export NGC_API_KEY="nvapi-..."

# Run the stack (NIM downloads the model on first run — ~5 min)
docker compose -f docker-compose.gpu-inference.yml up --build

# Watch for inference results
docker compose -f docker-compose.gpu-inference.yml logs -f miner-1 | grep GpuMiner
```

Expected output after ~2 epochs:
```
[GpuMiner] epoch=1 tokens=47 latency=1203ms model=meta/llama-3.2-1b-instruct
[GpuMiner] epoch=2 tokens=31 latency=892ms model=meta/llama-3.2-1b-instruct
```

## How it works

### Miner (`protocol.py` — `GpuInferenceProtocol`)

Each epoch:
1. Generate RA-TLS cert + TEE quote (binds cert pubkey into hardware-signed report_data)
2. Publish quote + cert to DHT
3. Pick a deterministic challenge prompt based on `sha256(epoch:peer_id)`
4. Call NIM's OpenAI-compatible API at `localhost:8000`
5. Wrap the response in an `OutputEnvelope` signed with the RA-TLS session key
6. Publish signed work record to DHT

### Validator (`protocol.py` — `GpuInferenceProtocol.validator_call`)

For each peer:
1. Fetch + verify TEE quote (7-step DCAP pipeline)
2. Fetch + verify RA-TLS cert (pubkey binding check)
3. Fetch + verify signed work record (OutputEnvelope signature)
4. Check: prompt matches expected challenge, completion is non-empty, latency < 30s

### Scoring (`protocol.py` — `GpuInferenceScoring`)

- All checks pass + real hardware → `score = 1.0`
- All checks pass + mock TEE → `score = 0.5`
- Any check fails → `score = 0.0`

## Integrating into your subnet

To use this as your subnet's node implementation:

```bash
# Copy the protocol to your subnet
cp examples/gpu-inference/protocol.py subnet/node/gpu_inference.py

# Update subnet/server/server.py to import GpuInferenceProtocol instead of MockNodeProtocol
# Update subnet/server/loops.py if needed
```

Or fork and modify `protocol.py` directly for your use case (different model, custom scoring, etc.).

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `NIM_BASE_URL` | `http://localhost:8000` | NIM API endpoint |
| `NIM_MODEL` | `meta/llama-3.2-1b-instruct` | Model name for NIM |
| `NIM_MODEL_PROFILE` | — | Set to `default` if NIM rejects your GPU |
| `NGC_API_KEY` | (required) | NVIDIA NGC API key for pulling NIM container |
| `MOCK_TEE` | `true` | Set `false` for real TEE attestation |
| `TEE_BACKEND` | — | Set `sev-snp` on Azure CVM for testing |
| `EXPECTED_MEASUREMENT` | — | Set to MRENCLAVE for production (Gramine/SGX) |

## Production notes

This example uses CVM (SEV-SNP) which is suitable for **testing** but not production.
For production deployment, use Gramine/SGX — see [GRAMINE.md](../../GRAMINE.md) and
[anti-cheat §10a](../../docs/04-anti-cheat.md#10a-runtime-code-tampering-inside-a-cvm).
